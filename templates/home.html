Results of the Current Model



This graph plot the models projected hours against the actual hours registered for the job.
For a perfect model, all points should be along the optimal line.

When making statements about how well the model is working, there are two things to talk about:
1. How well the actual data trends toward the optimal? What is the direction of the data?
2. How spread out is the model? How close are points to the model?

This graph plot the average models projected hours against the actual hours registered for the job. This means at any given "Actual Result", a red point is assigned to the average value given to the all jobs that registered that result
The blue dots are also calculated from all jobs registering a specific time result, however they plot the standard deviation above the average. (The standard deviation is useful because it signifies the spread of the data, a std of 20 says that 70% of data lies within +/- 20 of the average).

This graph is then used to summarize the data in the graph above by limiting the number of points shown. Therefore the same trends should follow for better models.
This graph is the standard bell curve used to describe most data. It describes the number of the times the difference of the data occured. Ideally there should only be one point where the difference between the actual time and the model's guess is zero.
Points on the right of the graph symbolize guesses that were made much lower than realistically speaking
Points on the left (negative numbers) occur when the model has guess a value higher than actuallity

The average and standard deviation are given above. Optimally the mean, or average should be zero, as well as the standard deviation (std).
